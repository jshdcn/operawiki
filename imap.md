# IMAP

Internet Message Access Protocol，是电子邮件领域的重要标准，主要由RFC 3501定义。

IMAP可以在服务器及客户端之间双向同步邮件和存放邮件的文件夹。几乎所有的主流邮件客户端，如[Opera邮件](m2.md)、[Thunderbird](thunderbird.md)和各大[操作系统](os.md)的自带客户端都支持IMAP。RFC 2177定义了一种IDLE状态，可以在新邮件来到时收到通知，相当于具有推送功能，桌面客户端一般支持。

IMAP的主要缺点是服务器和本机端都需要同样大小的存储空间，对带宽和电量的消耗也比较大。近年来在邮件推送市场，特别是移动端遇到了[Exchange](exchange.md)协议的强力挑战。
