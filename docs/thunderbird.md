# Mozilla Thunderbird

Mozilla Thunderbird是脱胎于Mozilla的两大商品化专项之一，重点负责邮件和新闻组功能的开发。最早面世于2003年。

Thunderbird的代码以MPL/GPL/LGPL三重授权发布，使用时可任选其一。例如它的衍生产品就多选择了LGPL以使自己的代码得到保护。

Thunderbird在战术上取得了巨大的成功，它迫使一大批邮件客户端停产倒闭，包括Netscape Messenger 9和Eudora 8在内的业界大腕纷纷放弃原有代码，投入雷鸟门下。就连[Mozilla基金会](mozilla.md)起初试图发展为独立日历软件的Sunbird，最终也化身为Lightning扩展被雷鸟收编。但是由于当时浏览器取代电邮客户端的趋势已经十分明显，Thunderbird在战略上一败涂地，其关注度远远低于同门的[Mozilla Firefox](firefox.md)。而在客户端占据重要地位的移动平台，雷鸟至今碌碌无为。

Thunderbird的版号管理十分与众不同，它的跳码现象非常严重，一般来说如果出现Firefox更新而Thunderbird没有同步跟上的话，Thunderbird的下一个版本号不是末位加1而是直接钉上新版Firefox。最著名的例子之一是Thunderbird 2的第一个版本不是2.0.0.0而是2.0.0.4。自从Firefox升3以后，Thunderbird继续维持2内核的开发，这种现象一度终结。但是[Gecko](gecko.md)版本号腾飞之后，Thunderbird再度跃迁，当然这只是升级内核版本，并非功能演进真正跟上Firefox。最后一次重量级的功能演进当属3.0和3.1版本，引入并完善了邮件全文搜索。

2022年，Thunderbird收编了Android平台上负有盛名的开源电邮客户端K-9 Mail而推出Thunderbird Mobile，版本号从13起跳。此举实质上也放弃了内核版本同Firefox的对齐。

长久以来Thunderbird都不支持移动端广为流行的Microsoft Exchange协议，这使其丢失了不少市场份额。从145版本有了转机，Thunderbird宣布通过EWS协议兼容Exchange服务器，并计划在未来升级到微软最新的Graph协议。
